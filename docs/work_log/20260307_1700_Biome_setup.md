# Biome導入作業記録

## 作業日時
2026年3月7日 17:00

## 概要
コードフォーマッターおよびリンターとしてBiomeを導入しました。BiomeはESLintやPrettierの代替となる高速なツールチェーンで、Rust製のため非常に高速です。

## 変更内容

### インストール
- `@biomejs/biome` v2.4.6 をインストール
- `--save-exact` フラグを使用して正確なバージョンを固定

### 設定ファイル
- `biome.json` を作成
  - VCS統合を有効化（gitignoreを使用）
  - `dist/` フォルダを除外
  - フォーマッター: タブインデント、ダブルクォート
  - リンター: 推奨ルールを有効化
  - インポートの自動整理を有効化

### package.json スクリプト追加
- `format`: コードフォーマットを適用
- `lint`: リンティングを実行・修正
- `check`: フォーマットとリンティングを両方実行・修正
- `prepare`: Huskyの初期化スクリプト

### Git Hooks設定
- `husky` v9.1.7 をインストール
- `lint-staged` v16.3.2 をインストール
- `.husky/pre-commit` を作成し、コミット前にlint-stagedを実行
- `lint-staged` 設定を追加
  - ステージングされたファイルに対してBiomeを自動実行
  - 自動修正が可能な問題は自動的に修正
  - Biomeが対応していないファイルは無視

### 自動修正された内容
Biomeを実行した結果、以下の問題が自動修正されました：
- 未使用のインポートを削除
- 未使用の変数をアンダースコアプレフィックス付きに変更
- 文字列連結をテンプレートリテラルに変換
- コードフォーマットの統一（インデント、クォートスタイルなど）

### 影響を受けたファイル
- `package.json`, `package-lock.json`: Biome、husky、lint-staged追加・スクリプト追加
- `biome.json`: 新規作成
- `.husky/pre-commit`: 新規作成（pre-commitフック）
- 全ソースファイル: フォーマット・リントによる自動修正
  - `src/analyzers/*.ts`
  - `src/cli/commands.ts`
  - `src/git/*.ts`
  - `src/reporters/*.ts`
  - `src/types/index.ts`
  - `src/utils/*.ts`
  - `tsconfig.json`, `tsup.config.ts`

## 使い方

### フォーマットのみ実行
```bash
npm run format
```

### リンティングのみ実行
```bash
npm run lint
```

### フォーマット + リンティングを実行
```bash
npm run check
```

### Git Hooks（自動実行）
コミット時に自動的にBiomeが実行されます。
- `git commit` を実行すると、ステージングされたファイルに対して自動的にBiomeがチェック・修正を実行
- 問題がある場合は、自動修正後に再度ステージングが必要
- エラーがある場合はコミットが中断されます

### 手動実行
必要に応じて手動でBiomeを実行することもできます。
- `npm run check`: プロジェクト全体をチェック・修正

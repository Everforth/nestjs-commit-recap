# DTOファイルの除外機能実装

## 作業日時
2026-03-21 18:35

## 概要
設計意思決定レポートからDTOファイルを除外する機能を実装しました。DTOファイルのみの変更は対象外とし、DTO + Entityなどの場合はDTOを除いた残りのファイルのみを対象とします。

## 変更内容

### 修正ファイル
- `src/design-decisions/data-collector.ts`

### 実装内容

#### 1. DTOフィルタリング用メソッド追加
```typescript
/**
 * DTOファイルを除外
 */
private filterOutDTOFiles(files: string[]): string[] {
  return files.filter((f) => {
    const lowerPath = f.toLowerCase();
    return !lowerPath.includes("dto");
  });
}
```

パス内に `dto` が含まれるファイルを除外します（大文字小文字を区別しない）。

#### 2. PRベースの処理を修正
`filterAndEnrichChanges` メソッド内のPR処理部分で:
- DTOファイルを除外
- DTOファイルのみの変更の場合はスキップ
- DTOを除いたファイルでカテゴリ分類
- DTOを除いたファイルの差分のみを含める

#### 3. コミットベースの処理も修正
コミット処理部分でも同じロジックを適用しました。

## 動作

### 除外パターン
以下のケースで動作します：

1. **DTOのみの変更** → 除外（targetChangesに含まれない）
   - 例: `["user.dto.ts"]`

2. **DTO + Entity** → Entityのみが対象
   - 例: `["user.dto.ts", "user.entity.ts"]` → `files: ["user.entity.ts"]`, `category: "domain-model"`

3. **Controller + DTO** → Controllerのみが対象
   - 例: `["user.controller.ts", "user.dto.ts"]` → `files: ["user.controller.ts"]`, `category: "api-endpoint"`

4. **Entity のみ** → そのまま対象
   - 例: `["user.entity.ts"]` → `files: ["user.entity.ts"]`, `category: "domain-model"`

### DTOファイルの判定基準
パス内に `dto` が含まれていればDTOファイルと判定します（大文字小文字を区別しない）。

以下のようなファイルが対象になります：
- `user.dto.ts`
- `src/dto/user.dto.ts`
- `user-dto.ts`
- `createUserDTO.ts`

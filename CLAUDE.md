# CLAUDE.md

このファイルはClaude Codeがプロジェクトで作業する際のガイドラインです。

## 作業前の調査

ライブラリの導入や外部APIの利用を伴う作業を行う際は、必ず最新情報を調べてから作業を開始してください。

- 公式ドキュメントで最新のAPI・設定方法を確認
- 破壊的変更やdeprecatedな機能がないか確認
- 推奨されるベストプラクティスを確認
- 関連する docs/work_log のドキュメントを確認


## 作業記録

作業を行う際は、必ず以下の形式で作業記録を作成してください。
**作業記録は必ずPRに含めてください。**

### ファイル名
```
docs/work_log/YYYYMMDD_HHmm_作業タイトル.md
```

例: `docs/work_log/20251215_2246_Neon_Drizzle_setup.md`

### 記録内容
- 作業日時
- 概要
- 変更内容（ファイル一覧と変更概要）
- 使い方（必要に応じて）
- 参考リンク（必要に応じて）

## TODO 管理

`docs/todos/` ディレクトリ内のTODOファイルを以下のルールで管理してください。

- **完了したタスクは削除する**: すべてのタスクが完了したTODOファイルは削除してください
- **進行中のタスクは更新する**: 一部のタスクが完了した場合、完了マーク（✅）を付けてファイルを更新してください
- **新しいタスクは追加する**: 新たな課題や実装予定の機能がある場合、適切なファイル名でTODOファイルを作成してください

例:
- `behavior-fixes.md` - 全タスク完了 → ファイルを削除
- `feature-additions.md` - 一部完了 → ✅ をつけて更新
- `new-feature-plan.md` - 新規タスク → 新規作成

## README 更新

技術スタックやセットアップ手順に変更があった場合、README.md を必ず更新してください。

## CLAUDE.md 更新

ユーザーからの指示で作業に新たなルールを指定された場合、必ず CLAUDE.md を更新してください。
